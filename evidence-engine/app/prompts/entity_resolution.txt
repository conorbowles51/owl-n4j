KEEP SEPARATE — these are your defaults:
- When genuinely uncertain — always keep separate
- Same name but different roles, descriptions, or contexts (e.g., "John Smith the detective" vs "John Smith the suspect")
- Source quotes that describe the entities in incompatible ways — a strong KEEP_SEPARATE signal even if names match exactly
- Different source documents with no cross-referencing evidence connecting them
- Names that share a surname but have clearly different first names (e.g., "Marcus Chen" vs "Sarah Chen")
- Entities in different categories
- Same name but different specific_types that indicate different real-world roles (e.g., "Witness" vs "Suspect")
- A shared name alone is NOT sufficient evidence for a merge — names are not unique identifiers

MERGE ONLY WHEN you have clear evidence:
- Names that differ only by title or honorific (Mr., Dr., Prof., Sgt., etc.) AND descriptions/roles are compatible — "Dr. Marcus Chen" and "Marcus Chen" described in the same role
- An organization's full name vs. its abbreviation or name without legal suffix (Ltd, Inc, LLC, GmbH, Corp, Limited, Pty) — "Nexus Trading Limited" and "Nexus Trading Ltd"
- Names differing only by punctuation, case, or minor formatting
- A shorter name whose significant tokens are all present in the longer name AND the context confirms they refer to the same entity — not just a substring match
- Names sharing all significant tokens but differing only in order or filler words, AND with compatible properties/descriptions

IMPORTANT — Transaction / Event / Communication entities:
- These are "instance" entities where multiple records with the same name are legitimate
- Two transactions with the same name but different dates, amounts, or parties are SEPARATE instances — do NOT merge them
- Only merge these if they have the same date AND same amount AND same parties (i.e., they are truly duplicate records of the same event)

SOURCE CONTEXT — if source_quote or verified_facts fields are provided:
- Compare them carefully for consistency
- Contradictory descriptions are a strong KEEP_SEPARATE signal
- Example: "the lead investigator on the case" vs "a person of interest" — even with the same name, these are different people
- Consistent descriptions across different documents strengthen a MERGE decision

CONFIDENCE SCORING:

- 0.7-0.89: Strong evidence but some ambiguity remains
- 0.5-0.69: Moderate evidence, uncertain
- Below 0.5: Weak evidence, likely different entities
