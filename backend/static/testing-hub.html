How to use this. Each item says what the feature is and how to test it. Set Pass / Fail / Blocked, add a note, and — if something's broken — fill in Reproduction steps so a dev can recreate it. Everything saves automatically under your name. Click an item's title or 💬 Discussion to open its thread and leave comments or updates; each comment is dated and attributed. You'll also see what other testers recorded, so it's clear who checked what. Tags show confidence: data verified = underlying logic already confirmed against the live case (mainly a visual check); needs your eyes = built & compiles but the on-screen behaviour hasn't been exercised — please test it properly. Most features live in: open a case → Open Workspace → Phone Reports → (device).
