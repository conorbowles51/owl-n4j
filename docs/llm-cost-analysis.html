Why Perplexity Doesn't Suit Owl's Ingestion

Every API call triggers a web search, adding a $5–$14 per 1,000 requests fee on top of token costs. Owl's ingestion extracts entities from your own documents — web search adds no value and inflates costs significantly.

For 464 files × ~35 calls = ~16,240 requests → $81–$227 in request fees alone, before any token costs.

Also missing: No embeddings API. No vision model. JSON mode reliability unproven for structured extraction at scale.

Where Perplexity Excels

Enterprise Pro ($40/seat/mo) is excellent for your investigation team's manual research — web-grounded answers with citations, SOC 2 compliant, data never used for training. Highly relevant for legal investigations.

Sonar API for geo-location rescan — the one ingestion task where web-grounded search genuinely adds value (verifying addresses, looking up property records).

4. Provider Profiles

OpenAI

$0.05 – $14.00 / 1M tokens
  • Pros: Native integration (zero code changes), full model range, excellent JSON mode, vision, embeddings, batch API (50% off)
  • Cons: Premium models are expensive
  • JSON mode: Native
  • Integration effort: None — change .env

Google Gemini

$0.10 – $10.00 / 1M tokens
  • Pros: Extremely cheap Flash models, excellent structured output support, generous free tier, vision support
  • Cons: Requires google-genai SDK or Vertex AI OpenAI-compat endpoint
  • JSON mode: Native
  • Integration effort: Moderate — new SDK or base_url

Anthropic Claude

$1.00 – $75.00 / 1M tokens
  • Pros: Top-tier quality for analysis and reasoning, excellent at following complex instructions, strong safety
  • Cons: No native JSON mode (uses tool_use), no embeddings, expensive, needs SDK adapter
  • JSON mode: Via tool_use
  • Integration effort: High — new SDK + adapter

xAI Grok

$0.20 – $15.00 / 1M tokens
  • Pros: Very competitive pricing on Grok 4.1 Fast, OpenAI-compatible API, structured output support
  • Cons: Newer platform with less production track record, no embeddings
  • JSON mode: Native
  • Integration effort: Low — base_url swap

Mistral AI

$0.10 – $6.00 / 1M tokens
  • Pros: Excellent price/performance ratio, strong JSON mode, OpenAI-compatible, EU-based (GDPR)
  • Cons: No vision models, no embeddings API
  • JSON mode: Native
  • Integration effort: Low — base_url swap

DeepSeek (Direct)

$0.14 – $2.19 / 1M tokens
  • Pros: Extremely cheap, competitive quality for structured tasks, cache discounts (90% off), off-peak discounts
  • Cons: China-based (data residency concerns for legal work), occasional availability issues
  • JSON mode: Native
  • Integration effort: Low — base_url swap

Together AI

$0.14 – $0.88 / 1M tokens
  • Pros: 200+ open-source models, strong JSON mode with schema enforcement, fine-tuning available, US-hosted
  • Cons: Quality varies by model, no proprietary frontier models
  • JSON mode: Native + Schema
  • Integration effort: Low — base_url swap

Groq

$0.06 – $0.99 / 1M tokens
  • Pros: Fastest inference (LPU hardware), cheapest per-token pricing, batch API (25% off), great for high-volume
  • Cons: Open-source models only, rate limits may constrain batch ingestion, no vision
  • JSON mode: Native
  • Integration effort: Low — base_url swap

Perplexity Sonar

$1.00 – $15.00 / 1M + request fees
  • Pros: Web-grounded answers with citations, excellent for research tasks requiring current information
  • Cons: Mandatory web search adds cost, no embeddings, no vision, no JSON mode guarantee, request fees

  • Integration effort: Low — base_url swap

5. Recommended Strategy

Tiered Task-Based Routing

Rather than switching everything to one provider, route each task to the most cost-effective model that meets its quality requirements. The Owl codebase already supports LLM_PROVIDER and LLM_MODEL configuration.

Tier 1: Bulk Extraction & Disambiguation — 93% Savings

Model: GPT-4o Mini ($0.15 / $0.60) or GPT-5 Mini ($0.25 / $2.00)

Tasks: Entity extraction, relationship extraction, disambiguation, processing estimate

Why: These tasks require reliable JSON output but don't need frontier-level intelligence. Mini models handle structured extraction very well.

Implementation: Change OPENAI_MODEL=gpt-4o-mini in .env — zero code changes.

Savings: $204 → $14 per 464-file batch

Tier 2: Quality-Sensitive Tasks — Keep Premium

Model: GPT-5.1 ($1.25 / $10.00) or GPT-5 Mini ($0.25 / $2.00)

Tasks: Entity summaries, RAG answer synthesis, investigation insights, defence analysis

Why: Prose quality and nuanced legal reasoning matter here. These are low-volume (on-demand) so cost impact is minimal.

Implementation: Add per-task model override in llm_client.py

Tier 3: Embeddings — Free Alternative

Model: Local sentence-transformers (all-MiniLM-L6-v2) or Ollama embeddings

Tasks: Document chunk embeddings for RAG vector search

Why: Eliminates embedding API costs entirely. Config already supports EMBEDDING_PROVIDER=ollama.

Implementation: Change EMBEDDING_PROVIDER=ollama in .env

Tier 4: Future Optimisation — Multi-Provider

Models: Gemini 2.0 Flash, Grok 4.1 Fast, DeepSeek V3, or Groq Llama 3.3 70B

Tasks: Bulk extraction at even lower cost ($3–$15 per 464-file batch)

Why: All support OpenAI-compatible endpoints. Requires adding base_url configuration and quality validation.

Implementation: Add provider routing in llm_client.py with base_url override

6. Quick Start — Immediate Savings

Option A: One-Line .env Change (93% savings, 30 seconds)

# In /owl-n4j/.env — change:
OPENAI_MODEL=gpt-5.1

# To:
OPENAI_MODEL=gpt-4o-mini

Option B: Per-Task Model Routing (maximum optimisation)

# In /owl-n4j/.env — add task-specific overrides:
OPENAI_MODEL=gpt-4o-mini          # Default for bulk extraction
OPENAI_SUMMARY_MODEL=gpt-5-mini   # For entity summaries
OPENAI_INSIGHT_MODEL=gpt-5.1      # For investigation insights / RAG
EMBEDDING_PROVIDER=ollama          # Free local embeddings

Perplexity Enterprise Pro — Separate Investment

Consider Enterprise Pro at $40/seat/month as a separate line item for your investigator team. It provides unlimited AI-powered research with citations, SOC 2 compliance, and data privacy guarantees — valuable for legal work, but separate from ingestion costs.
