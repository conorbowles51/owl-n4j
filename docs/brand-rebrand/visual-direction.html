Visual direction

Dense, source-led, and quietly premium.

The visual system should keep the current product's operational density, but shift the identity away from mascot cues and generic AI styling. The signature language should be source panels, binding lines, evidence nodes, citations, and restrained casework surfaces.

Palette

Refined investigation palette

Keep slate and brass from the current product's serious investigative feel, but add teal as the source/provenance accent and off-white paper as a humane counterweight.

Typography

Functional first

UI
Keep an Inter-like sans for dense tables, filters, sidebars, and graph panels. The current product already benefits from compact, familiar UI type.
Marketing
Use a warmer humanist sans such as IBM Plex Sans or Source Sans 3. It reads serious without feeling sterile.
Evidence IDs
Use a restrained monospace for file hashes, evidence IDs, job IDs, and source references.
Avoid
Oversized display type inside product panels, condensed techno wordmarks, and decorative serif typography.

Logo system

Source marks, not mascots

Primary motif
Binding spine plus source nodes, document frames, bracketed source paths, or citation points.
Construction
Geometric, flat, high-contrast, strong at 16px to 48px, and simple enough for app nav.
Wordmark
Title case or uppercase should be tested. Avoid overly wide tracking or sci-fi letterforms.
Avoid
Owls, eyes, search icons, shields, beams, courtroom gavels, detective hats, and AI sparkles.

UI direction

How the product should look after rebrand

Navigation

Keep dense left navigation and case-specific modes. Rename AI surfaces around source-cited work, for example Source Chat, Source Agent, Evidence Explorer, Case Graph, Timeline, Map, Financial, Reports.

Source panels

Make source trails a visual signature. Citations should look like first-class objects: source file, quote, extraction confidence, entity link, and review status.

Graph language

Use existing category colors inside graph views, but reserve teal and brass for provenance, selection, and reviewed findings so brand and data colors do not fight.

AI surfaces

Use AI language sparingly. Prefer "source-backed answer", "artifact", "review", "citation", and "confidence" over assistant personality language.

Implementation notes

What to change next

Brand migration

Audit product strings that still use the current placeholder name. Replace visible naming only after the legal choice is made.

Identity production

Have a designer rebuild the chosen mark in vector form, test it at 16px, create light/dark/mono variants, and produce favicon/app-icon assets.

Website design

Lead with actual product views and source trail proof. Avoid abstract illustrations when customers need to inspect credibility.
