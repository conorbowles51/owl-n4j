Add folder/document summaries, file preview improvements, and case backup/restore system

Major Features:
- Folder and Document Summaries:
  - Added AI-generated summaries for processed folders and documents
  - Folder summaries displayed above "File Statistics" in FileInfoViewer
  - Document summaries displayed when clicking on files
  - Summaries stored in both Neo4j and Vector DB for semantic search
  - Backend endpoints: GET /api/evidence/summary/{filename} and GET /api/evidence/folder-summary/{folder_name}
  - Frontend components: DocumentSummaryDisplay and FolderSummaryDisplay

- File Preview Enhancements:
  - Replaced toggle button with tabbed interface (Content and AI Summary tabs)
  - AI Summary tab is default when summary is available, otherwise Content tab
  - Improved visual design with proper tab styling
  - Made entire file row clickable in case view (not just eye icon)
  - Added selection highlighting for selected files (blue background and border)

- Case Backup and Restore System:
  - Complete backup/restore functionality for cases
  - Backup exports: Neo4j nodes/relationships, Vector DB documents/entities, case metadata, evidence records
  - Backup format: ZIP file containing backup.json and optional file contents
  - Restore functionality with overwrite protection
  - Backend service: BackupService with export_case() and import_case() methods
  - Backend endpoints: GET /api/cases/{case_id}/backup, POST /api/cases/{case_id}/restore
  - Frontend UI: Backup and Restore buttons in case header with loading states

Bug Fixes:
- Fixed generate_cypher_from_graph() missing case_id argument in wiretap processing
- Removed deprecated cypher_queries parameter from save_case_version() calls
- Fixed missing Download and HardDrive icon imports in CaseManagementView

UI/UX Improvements:
- Removed "Backfill Database" button from Evidence Files section in case view
- Improved file selection UX with full-row clickability and visual feedback
- Enhanced FilePreview with cleaner tab interface
- Better loading states and error messages for backup/restore operations

Backend Changes:
- Added backup_service.py with comprehensive case data export/import
- Enhanced neo4j_service.py with get_folder_summary() method
- Updated evidence.py router with folder summary endpoint
- Fixed wiretap processing to remove Cypher generation (data persists in Neo4j)
- Added proper case metadata import in restore functionality

Frontend Changes:
- Added DocumentSummaryDisplay and FolderSummaryDisplay components
- Updated FileInfoViewer to show summaries for files and folders
- Enhanced FilePreview with tabbed interface
- Added backup/restore handlers and UI in CaseManagementView
- Updated casesAPI with backup() and restore() methods
- Improved file list selection with full-row clickability
